## 📌 Описание проекта

**AI-Traffic Perm** — это интеллектуальная программная платформа верхнего уровня, предназначенная для предиктивного координированного управления светофорными объектами в крупных агломерациях (на примере контура из 45 ключевых перекрестков г. Перми). 

В отличие от классических адаптивных АСУДД, считающих «штуки» абстрактных машин, данная платформа переводит управление в **Human-Centric** парадигму: оптимизация транспортной сети рассчитывается в минимизации задержек пассажиропотока (**в человеко-часах**) за счет предоставления жесткого предиктивного математического приоритета общественному транспорту.

## 🏗️ Архитектура технологии

Платформа представляет собой распределенную ИИ-систему, разделенную на три стековых уровня вычислений:

1. **Edge Data Acquisition (Периферийный сбор):** Компьютерное зрение (YOLOv8-nano) на дорожных камерах фиксирует **длину очередей** перед стоп-линиями. Данные обогащаются по API (Apache Kafka) телеметрией ГЛОНАСС/GPS от навигационных систем автобусных парков для точной классификации типов транспорта.
2. **Graph Fusion Level (Графовая агрегация):** Дорожная сеть города преобразуется в пространственно-временной граф. **Графовые нейросети внимания (GAT)** агрегируют эмбеддинги состояний смежных перекрестков, прогнозируя развитие заторов на 15–20 минут вперед и устраняя «эхо-эффект» лавинообразных пробок.
3. **Multi-Agent Decision Level (Принятие решений):** Каждый перекресток управляется независимым ИИ-агентом на базе многоагентного глубокого обучения с подкреплением (**Multi-Agent RL: MAPPO + QMIX**). Агент итерационно каждые 5 секунд рассчитывает оптимальную светофорную фазу.

### 🧮 Функция вознаграждения ИИ (Reward Function)

Обучение агентов базируется на максимизации награды (минимизации штрафа за заторы) с повышенным коэффициентом приоритета для автобусов:

\[R_t = - \sum_{n=1}^{N} \left( w_{\text{car}} \cdot L_{\text{car}, n} + w_{\text{truck}} \cdot L_{\text{truck}, n} + w_{\text{bus}} \cdot L_{\text{bus}, n} \right) - \alpha \cdot \text{Penalty}_{\text{switch}}\]

* Где L — длина очереди класса транспорта на полосе n.
* Весовые коэффициенты приоритета: \(w_{\text{car}} = 1.0\), \(w_{\text{truck}} = 1.5\), **\(w_{\text{bus}} = 5.0\) (пятикратный приоритет автобусов)**.
* \(\text{Penalty}_{\text{switch}}\) — штрафной демпфер против хаотичного мигания сигналов.

## 🛠️ Развертывание окружения и запуск

Инструкция написана для операционной системы Windows.

### Шаг 1: Установка физического движка SUMO
1. Скачайте и запустите установщик [SUMO msi](https://eclipse.dev).
2. Добавьте корневую папку симулятора в системные переменные среды:
   * **Имя переменной:** `SUMO_HOME`
   * **Значение:** `C:\Program Files\Eclipse\Sumo` (или ваш путь установки).
3. Добавьте в системную переменную `Path` путь к исполняемым файлам: `C:\Program Files\Eclipse\Sumo\bin`.

### Шаг 2: Настройка Python-окружения
Откройте терминал в папке проекта и выполните команды:

```bash
# Создание и активация изолированного виртуального окружения
python -m venv its_env
its_env\Scripts\activate

# Обновление pip и установка ИИ-библиотек
pip install --upgrade pip
pip install numpy gymnasium pettingzoo torch
```

### Шаг 3: Запуск итерационного цикла ИИ
Для проверки работоспособности интеграции Gymnasium-среды с микромоделированием в SUMO через интерфейс TraCI запустите скрипт:

```bash
python src/environment.py
```

При успешном запуске в консоль начнут поступать матрицы реальных длин очередей с дифференциацией автобусов и расчеты штрафных очков (Reward) нейросети для каждой итерации шага.
